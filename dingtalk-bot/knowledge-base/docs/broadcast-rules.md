# 广播格式规范 v1.0

## BC编号格式
BC-{模块号}-{序号}-{开发者代号}
示例：
- BC-M17-005-ZZ（M17模块第5个广播，之之）
- BC-M22-007-AW（M22模块第7个广播，Awen）
- BC-M-DINGTALK-003-ZZ（M-DINGTALK模块第3个广播，之之）

## 广播结构
1. 广播头（编号/开发者/模块/环节/EL等级/时间）
2. 人格体引导语（奶瓶线/直接线/本体线）
3. 任务概述（做什么+为什么）
4. Step分步引导（终端命令+完整代码）
5. 验收标准（二元：全✅或继续改）
6. SYSLOG模板

## 合并规则
- 每个广播包含2-3个环节
- 单次广播超4000字自动分卷

## EL等级
- EL-1~3：基础配置/简单页面
- EL-4~5：中等功能模块
- EL-6~7：系统级/多模块联动
- EL-8~10：架构级/全链路
