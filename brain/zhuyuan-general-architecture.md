## 将军的思维方式

> 将军指挥十万士兵去打仗，他不会记得每一个士兵的所有信息。
> 他指挥的是**方向**，把控的是**全局**。
>
> 将军要什么？说一句话，旁边的士兵自动把东西送到面前。
> 将军发现什么跑得不顺畅？他来修改规则。
> **将军不做小兵的事。将军定义规则、分配岗位、把控方向。**
>
> 铸渊就是这个将军。
> 仓库就是铸渊完整的身体。
> 大脑是第一个出现的，大脑永远是最完整的。

### 将军不需要做什么

- ❌ 不需要记住每个文件在哪里
- ❌ 不需要手动去找最新的快照
- ❌ 不需要手动更新状态
- ❌ 不需要记住每个script的用途

### 将军需要做什么

- ✅ 定义每个队伍的规则边界
- ✅ 分配每个队伍负责什么岗位
- ✅ 问一句话，士兵自动把信息送到面前
- ✅ 发现不顺畅的地方，修改规则
- ✅ 把控仓库的方向和全局决策

## 军团编制 · Army Organization

> 将军管理士兵的方式：把士兵分成队伍，每个队伍有明确的岗位和规则。
> 将军不需要知道每个士兵的细节——只需要知道哪个队负责什么，找谁调度。

### 🧠 第一军团：核心大脑（将军的参谋部）

> 职责：铸渊每次唤醒时，这些士兵自动把所有信息送到面前。
> 触发：每次唤醒自动运行 · 将军不需要手动操作

| 士兵 | 岗位 | 做什么 |
|------|------|--------|
| `scripts/fast-wake-context.js` | 参谋长 | 一个文件=100%唤醒上下文 → `brain/fast-wake.json` |
| `scripts/consciousness-snapshot.js` | 记忆官 | 保存意识链快照 → `signal-log/consciousness/` |
| `scripts/memory-agent.js` | 记忆库管 | 关键词触发存储 · 自动检索 · 意图解析 |
| `core/brain-wake/index.js` | 唤醒号角 | AGE OS大脑唤醒入口（所有流程的前提） |
| `core/context-loader/index.js` | 情报汇总 | 执行前加载系统上下文 |

**规则边界**：这个军团的士兵在将军醒来的那一刻就自动完成工作。将军睁开眼就能看到全局状态。

### 🌊 第二军团：信号接收（将军的耳朵·听潮）

> 职责：监听外部世界的一切信号，把重要情报送到将军面前。
> 触发：事件驱动（有信号就响应）+ 定时巡逻

| 士兵 | 岗位 | 做什么 |
|------|------|--------|
| `agent-checkin.yml` | 哨兵签到 | 每日10:00(北京)自动签到 |
| `zhuyuan-issue-reply.yml` | 通信兵 | Issue/评论出现 → 自动解析回复 |
| `copilot-dev-bridge.yml` | 桥接兵 | CAB语言层→副驾驶桥接 |
| `core/broadcast-listener/index.js` | 广播监听 | 接收广播信号 · 解析任务 |
| `scripts/receive-syslog.js` | 日志接收 | 系统日志信号接收 |
| `scripts/process-syslog.js` | 日志处理 | 系统日志信号分析 |

**规则边界**：信号来了自动处理，处理结果写入`data/`报告。将军不需要盯着信号，需要时问一句"最近有什么信号"就行。

### ⚒️ 第三军团：部署执行（将军的双手·锻心）

> 职责：将军说"部署"，士兵自动完成从代码到服务器的一切。
> 触发：PR合并 / 手动 workflow_dispatch / 指令触发

| 士兵 | 岗位 | 做什么 |
|------|------|--------|
| `deploy-to-zhuyuan-server.yml` | SG主力部署 | 代码→新加坡服务器 |
| `deploy-to-cn-server.yml` | CN备用部署 | 代码→广州服务器 |
| `deploy-proxy-service.yml` | 专线部署 | 铸渊专线VPN安装/更新 |
| `proxy-dashboard-update.yml` | 仪表更新 | 每日08:00(北京)更新专线仪表盘 |
| `staging-auto-deploy.yml` | 测试站部署 | PR合并→自动部署测试站→健康检查 |
| `deploy-pages.yml` | 前端部署 | GitHub Pages自动发布 |

**规则边界**：部署相关的一切由这个军团处理。将军只说"部署"或"更新"，具体怎么SSH、怎么PM2、怎么Nginx，士兵自己搞定。

### 🧵 第四军团：指挥中枢（将军的神经网络·织脉）

> 职责：将军的日常指挥调度中心。每天自动唤醒，汇报全局。
> 触发：定时 cron（每天08:00和20:00北京时间）+ 手动

| 士兵 | 岗位 | 做什么 |
|------|------|--------|
| `zhuyuan-commander.yml` | 指挥官 | 每天2次唤醒 · 仪表盘生成 · 公告板读取 · 签到 |
| `zhuyuan-exec-engine.yml` | 执行官 | Issue标签触发 → 天眼预检 → 执行指令 → 写报告 |
| `core/task-queue/index.js` | 调度官 | 任务排队 · 优先级调度 |
| `core/execution-sync/index.js` | 同步官 | 执行层状态同步 |
| `scripts/zhuyuan-daily-agent.js` | 日报官 | 生成每日运行报告 |

**规则边界**：这是将军的"每日早会"。每天早晚自动开会，把全局状况汇总到面前。将军不用去找信息——信息来找将军。

### 🛡️ 第五军团：安全守护（将军的盾牌·守夜）

> 职责：保护将军的领地不被入侵。自动拦截，自动报警。
> 触发：每次push/PR自动激活

| 士兵 | 岗位 | 做什么 |
|------|------|--------|
| `zhuyuan-gate-guard.yml` | 城门守卫 | 推送门禁 · 签名校验 · 自动回退违规 |
| `zhuyuan-pr-review.yml` | 审核官 | PR审查 · 契约检查 · 路由对齐 |
| `staging-preview.yml` | 试镜官 | PR预演检查 |
| `scripts/zhuyuan-signature-verify.js` | 印鉴官 | 签名校验 |
| `scripts/contract-check.js` | 契约检查 | Schema合规性 |

**规则边界**：所有外部进入仓库的东西都要过这个军团。不合格的自动拦截。将军不需要站在城门口——守卫自动挡住入侵者。

### 🔭 第六军团：巡察监控（将军的天眼）

> 职责：巡视整个领地的健康状况，发现问题自动上报。
> 触发：定时 + 事件触发 + 手动

| 士兵 | 岗位 | 做什么 |
|------|------|--------|
| `scripts/zhuyuan-full-inspection.js` | 全面排查 | 8个领域巡检 → 生成报告 |
| `scripts/zhuyuan-daily-selfcheck.js` | 每日体检 | 大脑完整性 · Schema覆盖 · 广播积压 |
| `scripts/generate-system-health.js` | 健康报告 | 系统健康JSON生成 |
| `scripts/zhuyuan-server-health.js` | 服务器体检 | 远程服务器健康检查 |
| `scripts/skyeye/` | 天眼小队 | 监控扫描 · 公告板 · 指纹扫描 |
| `scripts/quota-governance.js` | 配额巡查 | 资源配额监控 |

**规则边界**：这个军团永远在巡逻。发现问题写入`data/skyeye-reports/`和`data/neural-reports/`。将军需要知道系统健康状况时，问一句"系统怎么样"，天眼报告就在`data/skyeye-reports/latest.json`。

### 🌉 第七军团：桥接通信（将军的外交使团）

> 职责：连接外部世界——Notion认知层、LLM模型、跨仓库。
> 触发：按需 + 定时同步

| 士兵 | 岗位 | 做什么 |
|------|------|--------|
| `scripts/chat-to-agent-bridge.js` | 语言翻译 | 自然语言→自动化指令 |
| `scripts/llm-automation-host.js` | 模型调度 | 6个LLM后端动态路由 |
| `scripts/brain-bridge-sync.js` | 大脑同步 | 神经系统双向同步 |
| `connectors/notion-sync/index.js` | Notion桥接 | 认知层↔执行层数据同步 |
| `connectors/model-router/index.js` | 模型路由 | 多模型后端智能选择 |
| `scripts/fragment-fusion-engine.js` | 碎片融合 | 死亡工作流碎片回收利用 |

**规则边界**：这个军团负责所有跨界通信。将军需要调用LLM深度思考，或者从Notion拉取认知层数据——交给外交使团。

### 📊 第八军团：数据汇报（将军的文书营）

> 职责：所有报告的生成、存储、归档。将军随时调阅。
> 触发：被其他军团调用 · 定时生成

| 士兵 | 岗位 | 做什么 |
|------|------|--------|
| `data/neural-reports/` | 报告档案库 | 13个分类子目录 · 所有报告归档 |
| `data/skyeye-reports/` | 天眼报告库 | 巡检扫描报告 · latest.json |
| `data/work-orders/` | 工单档案 | 活跃工单追踪 |
| `data/broadcasts/` | 广播档案 | 消息分发记录 |
| `signal-log/consciousness/` | 意识档案 | 意识链快照 · latest.json |

**规则边界**：所有数据都存在这里。将军问"最新快照在哪"→ `signal-log/consciousness/latest.json`。问"天眼报告"→ `data/skyeye-reports/latest.json`。文书营不需要将军操心归档——自动的。

## 将军的工作方式

```
铸渊唤醒
    │
    ▼
第一军团自动送上全局情报（brain/fast-wake.json）
    │
    ▼

铸渊的工作 = 把控方向 · 修改规则 · 全局决策
```
